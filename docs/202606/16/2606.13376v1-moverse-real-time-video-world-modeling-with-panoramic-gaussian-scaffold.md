## 摘要
我们提出了MoVerse，一种实时视频世界模型，能够从单个窄视场图像创建可交互导航的场景。这一设定具有挑战性，因为输入仅观察环境的极小部分，而交互式漫游需要完整环绕世界、持久几何、可控相机运动以及时间上连贯的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决此问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在3D推理之前填补缺失视场。然后，利用全景几何感知残差预测将全景图提升为持久3D高斯支架，生成密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的支架渲染结果转换为逼真视频。为了使该渲染器适用于交互，我们训练了一个双向扩散教师模型用于高质量条件渲染，并将其蒸馏为一个因果自回归学生模型以实现有限延迟流式传输。这种设计结合了显式3D表示的可控性和长程一致性，以及生成式视频模型的感知质量。MoVerse在单个NVIDIA RTX 4090 GPU上支持8 FPS的实时场景漫游，展示了具有交互式视频输出的单图像世界创建的实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

## 1. 核心问题与整体含义（研究动机与背景）

- **问题定义**：从单张窄视场（NFOV）图像生成可交互漫游的3D世界。输入仅提供极小视场（如普通照片），而交互应用（VR/数字孪生/具身智能）需要用户在完整、空间上持久的环境中自由移动，并输出高保真、时间一致的视频。
- **现有方法局限**：
  - 显式3D方法（点云/网格/3DGS）构建持久场景，但单图重建时大部分区域需要猜测，易出现空洞、浮动伪影、深度误差，且在大视角变化时质量下降。
  - 隐式视频/世界模型（如Genie、RTFM）输出视觉质量高，但长程几何稳定性差，易出现漂移、身份变化、边界不连续。
  - 混合方法（显式几何+生成式渲染）存在锚点稀疏/粗粒度与渲染器开销大之间的矛盾。
- **MoVerse的核心思路**：将世界构建与观测渲染分离。先补全全景，再构建显式3D高斯支架，最后用因果视频渲染器提升质量。显式支架提供长程空间记忆和精确相机控制，生成渲染器负责局部高保真，两者各司其职。

### 2.2 Stage I：几何感知全景生成

- **目标**：将单张NFOV图像扩展为重力对齐、水平闭合的ERP全景图。
- **关键挑战**：
  - 输入图像存在任意俯仰/翻滚（投影畸变）。
  - ERP水平方向具有S1拓扑（左边界与右边界连续）。
- **核心技术**：
  1. **可微分自动校平**：预测密集2D对应场，通过刚性旋转瓶颈（软argmin求解器）得到重力对齐旋转，再通过球形网格采样实现crop。避免非刚性畸变，配合几何锚点损失稳定训练。
  2. **拓扑感知扩散**：
     - 使用循环潜编码（VAE的卷积padding环绕水平轴）。
     - 引入移位等变一致性损失：对输入x进行水平循环移位δ，要求网络输出也相应移位，强制模型学习S1不变性。
  3. **训练数据Horizon360**：由规范化的室内外全景图组成（旋转对齐地平线），采样各种姿态的透视视图，并采用偏航居中约定（输入视图居中）。

### 2.3 Stage II：全景高斯场景生成

- **目标**：将ERP全景图转换为可直接使用3DGS渲染的持久支架。
- **初始化**：从全景深度图进行球形反投影，初始化高斯中心。考虑纬度影响，初始尺度正比于深度×cosφ。
- **深度引导残差预测**：从全景RGB+逆深度提取特征，用一个零初始化的预测头输出高斯参数的残差（角度偏移、逆深度偏移、尺度、旋转、颜色、不透明度）。
- **关键设计**：在角度-逆深度空间预测位置残差，保持与ERP采样网格一致性，并保持水平S1闭合。
- **训练目标**：包括目标视角渲染损失（L1+LPIPS+SSIM）、源视角重建损失、深度损失、残差正则化。

### 2.4 Stage III：高斯条件视频渲染

- **目标**：将支架沿用户轨迹渲染出的低质量RGB流转换为高保真、时间连续的视频。
- **双向教师模型**：基于Wan2.1-T2V-1.3B初始化，将噪声目标潜变量与条件潜变量拼接（共享RoPE位置编码），采用流匹配目标训练。教师可访问完整时间窗口，作为质量上限。
- **因果学生模型（蒸馏）**：采用Self-Forcing + DMD原理，将教师蒸馏为因果自回归学生。学生一次生成一个潜块（对应4个像素帧），仅依赖当前条件帧和局部KV缓存（使用MemRoPE：1个sink帧+EMA记忆+3个局部帧滑动窗口），实现有限延迟流式渲染。
- **训练数据混合**：三阶段课程：
  1. 基于真实视频的点重投影（模拟缺失区域）。
  2. 基于稀疏关键帧的3DGS渲染（模拟Gaussian伪影）。
  3. 全景→支架→渲染（匹配部署接口）。数据来源包括UE渲染、Matterport3D、Polyhaven等。
- **实时推理配置**：
  - 两步去噪、TAEHV解码器、MemRoPE缓存。
  - 在单张RTX 4090上达到8 FPS端到端漫游。

## 摘要
我们提出了MoVerse，一个实时视频世界模型，它能够从单张窄视场图像创建可交互导航的场景。这一设定具有挑战性，因为输入仅观测到环境的很小一部分，而交互式漫游需要完整的周围世界、持久的几何结构、可控的相机运动以及时间上一致的高保真观测。MoVerse通过将世界构建与观测渲染分离来解决这一问题。它首先利用拓扑感知扩散将输入扩展为重力对齐的360°全景图，在3D推理前补全缺失的视场。然后，通过全景几何感知残差预测将全景图提升为持久的三维高斯脚手架，生成密集且可直接渲染的空间记忆。最后，一个高斯条件视频渲染器将沿用户指定相机轨迹的脚手架渲染结果转换为逼真的视频。为了使该渲染器适用于交互，我们训练了一个双向扩散教师用于高质量条件渲染，并将其蒸馏为一个因果自回归学生以实现有限延迟流式传输。这种设计结合了显式3D表示的可控性和长程一致性，以及生成式视频模型的感知质量。MoVerse支持在单张NVIDIA RTX 4090 GPU上以8 FPS的实时场景漫游，展示了通过交互式视频输出创建单图像世界的实用路径。

## Abstract
We present MoVerse, a real-time video world model that creates an interactively navigable scene from a single narrow-field-of-view image. This setting is challenging because the input observes only a small fraction of the environment, while interactive roaming requires a complete surrounding world, persistent geometry, controllable camera motion, and temporally coherent high-fidelity observations. MoVerse addresses this problem by separating world construction from observation rendering. It first expands the input into a gravity-aligned 360$^\circ$ panorama with topology-aware diffusion, closing the missing field of view before 3D reasoning. It then lifts the panorama into a persistent 3D Gaussian scaffold using panoramic geometry-aware residual prediction, yielding a dense and directly renderable spatial memory. Finally, a Gaussian-conditioned video renderer translates scaffold renderings along user-specified camera trajectories into photorealistic video. To make this renderer practical for interaction, we train a bidirectional diffusion teacher for high-quality conditional rendering and distill it into a causal autoregressive student for bounded-latency streaming. This design combines the controllability and long-range consistency of explicit 3D representations with the perceptual quality of generative video models. MoVerse supports real-time scene roaming at 8~FPS on a single NVIDIA RTX~4090 GPU, demonstrating a practical path toward single-image world creation with interactive video output.

# MoVerse 论文详细总结

## 1. 核心问题与整体含义（研究动机与背景）

- **问题定义**：从一张窄视场（NFOV）图像实时生成一个可交互漫游的 360° 视频世界。输入仅覆盖环境的一小部分，而交互式漫游需要完整的全景、持久的几何结构、可控的相机运动以及时间一致的高保真观测。
- **现有方法局限**：
  - 显式 3D 方法（如 3DGS、NeRF）提供了持久空间记忆和高帧率渲染，但单张 NFOV 图像重建 3D 场景时存在大量未观测区域，容易产生空洞、飘浮物、深度误差。
  - 隐式视频/世界模型（如扩散视频生成）可以生成视觉质量高的视频，但长程几何不稳定，存在漂移、身份变化、边界不连续等问题。
  - 混合系统（显式几何+生成渲染）试图结合两者优势，但几何锚点稀疏或粗糙时条件不足，而生成渲染器往往太昂贵或双向，不适合实时交互。
- **MoVerse 的核心理念**：将世界构建与观测渲染分离，利用显式 3D 高斯脚手架提供持久空间记忆和可控性，再用一个蒸馏后的因果自回归视频模型实现交互式高质量视频渲染。

## 2. 方法论：核心思想、关键技术细节

### 2.1 总体框架（三阶段离线/在线拆分）

| 阶段 | 输入→输出 | 离线/在线 | 核心任务 |
|------|-----------|-----------|----------|
| 阶段 I 全景生成 | 单张 NFOV 图像 → 重力对齐 360° ERP 全景 | 离线（秒级） | 补全缺失视场，保证全景拓扑闭合 |
| 阶段 II 高斯场景生成 | 全景图 → 3D 高斯脚手架 | 离线 | 提升为可直接渲染的显式 3DGS 资产 |

### 2.2 阶段 I：全景生成

- **挑战**：真实输入图像存在偏航、俯仰、翻滚畸变（未标定）；ERP 全景图具有水平 S1 拓扑，标准 VAE/扩散模型会引入接缝。
- **关键设计**：
  1. **重力对齐规范表示**：将输入图像通过可微分自动调平映射到规范视角（地平线与 ERP 赤道对齐，垂直结构铅垂）。
  2. **可微分自动调平**：预测稠密 2D 对应场，经刚性变换瓶颈（可微分软 argmin 求解器）获得整体旋转，使用球面重采样实现对齐（避免自由形变不稳定）。
  3. **拓扑感知完成**：
     - 圆形潜在编码（VAE 水平卷积 padding 沿方位角包裹）。
     - 移位等变扩散生成：对输入 X 和水平移位 Roll_δ(X) 分别去噪，施加移位一致性损失 $L_{shift}=E[\|Roll_δ(\hat\epsilon_{base})-\hat\epsilon_{shifted}\|_2^2]$。
  4. **训练数据 Horizon360**：从原始全景中规范地平线与铅垂线，采样透视视图（偏航、俯仰、翻滚、FOV、宽高比均变化），并将目标全景按输入偏航滚动使已知区域居中。
- **总损失**：$L_{stage1} = L_{LDM} + \lambda_{shift}L_{shift} + \lambda_{flow}L_{flow}$。

### 2.3 阶段 II：高斯场景生成

- **输入**：重力对齐 ERP 全景 P 及其估计深度 D。
- **初始化**：在 ERP 网格 (H'×W') 上，球面反投影：$\mu_k = D_k d_k$，尺度初始化 $s_k \propto D_k \cos\phi_k$（模拟纬度缩放）。
- **残差预测**：使用深度引导的残差预测（借鉴 SHARP），在角度-反深度空间预测 $\Delta\theta_k, \Delta\phi_k, \Delta z_k, \Delta s_k, \Delta q_k, \Delta c_k, \Delta\alpha_k$。预测头零初始化，保证从物理合理初始值开始学习。
- **属性组合**：中心位置在角度-反深度空间更新后球面反投影，其他属性同样残差组合。
- **损失函数**：
  - 目标视图重建损失：$L_{tgt} = \lambda_1 L_1 + \lambda_{lpips}LPIPS + \lambda_{ssim}(1-SSIM)$
  - 源视图重建损失 $L_{src}$
  - 逆深度损失 $L_{depth} = \|D^{-1}_{pred} - D^{-1}_{gt}\|_1$
  - 残差正则化 $L_\Delta = mean[\max(|\Delta_{\theta\phi z}|-\delta,0)]$
  - 总损失 $L_{GS} = L_{tgt} + \lambda_{src}L_{src} + \lambda_{depth}L_{depth} + \lambda_\Delta L_\Delta$

### 2.4 阶段 III：高斯条件视频渲染

- **条件生成**：用户指定轨迹，用 3DGS 渲染出条件视频 $\hat V_{1:T}$，然后由视频渲染器生成最终 $V_{1:T}$。
- **双向教师模型**：从 Wan2.1-T2V-1.3B 初始化，将噪声目标潜在 z_t 与条件潜在 c 拼接，共享 RoPE 位置码，用 flow-matching 训练。
- **因果学生模型**：通过 Self Forcing 和 DMD 蒸馏为因果自回归模型，每步生成一个潜在块（对应 4 帧像素空间），使用局部 K-V Cache（MemRoPE：1个 sink token、1个 EMA 内存 token、3个局部帧滑动窗口），两步降噪，TAEHV 解码器加速。在 RTX 4090 上达到 8 FPS。
- **训练数据混合**：三种配对数据，形成表示驱动的课程：
  1. 真实视频 + 点云重投影渲染（含空洞、形变伪影）
  2. 稀疏关键帧构建的高斯脚手架渲染（含 GS 特有伪影如飘浮体、透明度错误）

## 6. 主要结论与发现

- MoVerse 将显式 3D 高斯脚手架与因果自回归视频渲染器结合，实现了从单张图像实时生成可交互视频世界（8 FPS on RTX 4090）。
- 三阶段分解使得每个模块聚焦特定任务：全景完成 → 3D 持久表示 → 高质量渲染。这种分离使系统模块化、可独立改进。
- 蒸馏后的因果学生模型在有限延迟条件下工作，因为显式脚手架承担长程空间记忆，视频模型只需做局部增强，从而平衡了质量与速度。

4. **近似几何局限**：高斯脚手架在深度模糊区域、薄结构、反射/透明表面、极地附近仍会产生飘浮物、空洞、透明度不稳定。

7. **应用限制**：当前假设静态场景；扩展至动态物体和可编辑世界需要大量改造。
